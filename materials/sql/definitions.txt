Задача 2024.11.08.03

От какого английского слова произошло слово реляционный?
Что оно означает?

relational произошло от слова relation, что означает отношение

В контексте баз данных, "реляционный" (relational) относится к следующему:

    - Реляционные базы данных: Это тип баз данных, где данные организуются в виде таблиц со связями между ними.
    - Отношения между таблицами: В реляционных базах данных данные представлены в виде таблиц, которые могут
      быть связаны друг с другом через ключи.
    - Структура данных: Реляционные базы данных используют структуру, основанную на отношениях между таблицами.

Реляционная модель данных - это модель представления и организации данных в базе данных.
В реляционной модели данные хранятся в виде таблиц, состоящих из строк и столбцов.
Каждая таблица имеет имя и структуру, определяемую набором столбцов и их типами данных.
И в этой модели данные из одной таблицы могут зависеть от данных в другой таблице.

Задача 2024.11.08.04

Что такое СУБД? Опишите вашими словами, человеческим языком.

СУБД - "Система управления базами данных". -- для управления большими объемами информации
Основные функции СУБД:
Хранение данных: СУБД похожа на огромный шкаф, где вы можете положить все ваши вещи. Она хранит информацию в такой
                 форме, чтобы ее можно было легко найти и использовать позже.
Организация информации: Внутри шкафа у вас есть специальные ящики, папки и коробки. СУБД использует подобную систему
                        для классификации и структурирования данных.
Поиск и доступ к информации: Когда вы ищете что-то в шкафу, СУБД помогает быстро находить нужную информацию,
                             даже если она хранится в разных местах.
Обеспечение безопасности: Шкаф имеет замок, чтобы защитить содержимое от несанкционированного доступа.
                          СУБД обеспечивает безопасность данных, предотвращая их случайное изменение или утечку.
Работа с несколькими пользователями: В реальной жизни несколько человек могут использовать один шкаф.
                                     СУБД позволяет многим пользователям одновременно работать с данными без конфликтов.

Задача 2024.11.08.11

Допишите в definitions.txt новые определения из полезных материалов. И дополните каждое своим пояснением своими словами
и примерами.

Сначала определение, потом своими словами: это для .. или это про ..., а потом примеры

* Реляционная модель данных - это модель представления и организации данных в базе данных. В реляционной модели данные
  хранятся в виде таблиц, состоящих из строк и столбцов. Каждая таблица имеет имя и структуру, определяемую набором
  столбцов и их типами данных. И в этой модели данные из одной таблицы могут зависеть от данных в другой таблице.

Определение:
Способ организации и представления данных в базах данных.

Примеры:
В библиотеке реляционная модель может представлять каталог книг как таблицу со столбцами "Название", "Автор", "Жанр".
Каждая строка в этой таблице - это отдельная книга.

----------------------------------------------------------------------------------------------------------------------

* Таблица - основной объект в БД.
  Она состоит из строк, столбцов и полей.
  Строки - это набор данных по горизонтали.
  Столбцы - это набор данных по вертикали.
  Поля - это названия столбцов.

Определение:
Основной способ организации данных в реляционной модели.

Примеры:
В базе данных таблица список книг в библиотеке со столбцами "ID", "Book Name", "Author".

----------------------------------------------------------------------------------------------------------------------

* Столбец - это одно поле в таблице, которое хранит данные одного типа. Каждый столбец имеет уникальное имя и тип
  данных, который определяет, какой тип данных может храниться в столбце.

Определение:
Вертикальная организация данных в таблице.

Примеры:
Название книги: Этот столбец содержит названия всех книг в библиотеке.
Автор: Этот столбец содержит имена авторов всех книг.

----------------------------------------------------------------------------------------------------------------------

* Строка - это одна запись в таблице, которая содержит данные для каждого столбца. Каждая строка в таблице имеет
  уникальный идентификатор, который называется ключом. Ключ может быть составным, т.е. может включать несколько
  столбцов.

Определение:
Горизонтальная организация данных в таблице.

Примеры:
В базе данных библиотеки строка может представлять одну книгу с данными по названию, автору, жанру и количеству
экземпляров

----------------------------------------------------------------------------------------------------------------------

* Ключ - это уникальный идентификатор для каждой записи в таблице. Ключ может быть составным, т.е. может включать
  несколько столбцов, которые вместе обеспечивают уникальность записи в таблице. Ключ используется для связывания
  данных в разных таблицах и для обеспечения быстрого доступа к данным.

Определение:
Уникальный идентификатор каждой строки в таблице.

Примеры:
Каждая книга имеет свой уникальный ID
